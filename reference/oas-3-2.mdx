Only deltas are documented â€” for base feature coverage, see the [OAS 3.0](/reference/oas-3-0) and [OAS 3.1](/reference/oas-3-1) pages.

### Server Object

| Field | Support | Behavior |
|-------|---------|----------|
| `name` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Server host identifier. Appears as "Server" in generated server docstring and metadata.json |

### Path Item Object

| Field | Support | Behavior |
|-------|---------|----------|
| `query` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | HTTP QUERY method (IETF draft-ietf-httpbis-safe-method-w-body). Generates an MCP tool identical to other HTTP methods â€” supports request bodies, parameters, security, and server URL overrides. When `operationId` is absent, auto-generated name uses `query_` prefix (e.g., `query_search`) |
| `additionalOperations` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Map of arbitrary HTTP method strings (e.g., COPY, MOVE, LOCK, PROPFIND) to Operation Objects. Each entry generates an MCP tool with the method string sent as-is (uppercased). Path-level `parameters` apply to all entries. Supports request bodies, response models, security, and server URL overrides |

### Parameter Object

#### New Parameter Location

| Location | Support | Behavior |
|----------|---------|----------|
| `querystring` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Treats the entire URL query string as a single parameter value, specified via `content` with a media type. Supported: `application/x-www-form-urlencoded`, `application/json` / `*+json`, `text/plain` / `text/*`. Unsupported media types fall back to `application/x-www-form-urlencoded`. MUST NOT coexist with `in: "query"` parameters; at most one per operation |

#### New Serialization Style

| Style | Support | Behavior |
|-------|---------|----------|
| `cookie` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | RFC 6265-compliant cookie serialization. Name-value pairs separated by `; `, no percent-encoding. Supports primitives, arrays, and objects with explode control. Default `explode` is `true`. Default `style` for `in: cookie` remains `form` for backward compatibility |

#### Changed Field Types

| Change | Support | Behavior |
|--------|---------|----------|
| `content` map values now allow Reference Object | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | `$ref` values in `content` maps (Parameter, Request Body, Response, Header) are resolved transparently. Works with `components/mediaTypes`, `components/headers`, and any local JSON Pointer target |

### Request Body Object

| Field | Support | Behavior |
|-------|---------|----------|
| `content` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Type changed to `Map[string, Media Type Object \| Reference Object]`. `$ref` values resolved transparently |

### Media Type Object

| Field | Support | Behavior |
|-------|---------|----------|
| `itemSchema` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Schema for each item in a sequential media type (JSONL, NDJSON, JSON-SEQ, SSE, multipart/mixed). Planned for future webhook/streaming support |
| `prefixEncoding` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Positional Encoding Objects for multipart types. Planned for future webhook/streaming support |
| `itemEncoding` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Per-item Encoding Object for multipart arrays. Planned for future webhook/streaming support |

<Note>
`encoding` MUST NOT be present if `prefixEncoding` or `itemEncoding` are present, and vice versa.
</Note>

#### Sequential Media Types

OAS 3.2 formalizes how sequential/streaming media types are modeled. These are treated as arrays in the JSON Schema data model; `itemSchema` describes each element. Planned for future webhook/streaming support.

| Media Type | Support | Behavior |
|------------|---------|----------|
| `application/jsonl` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | JSON Lines â€” each line is a JSON value |
| `application/x-ndjson` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Newline-Delimited JSON |
| `application/json-seq` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | JSON Text Sequences (RFC 7464) |
| `text/event-stream` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Server-Sent Events (SSE) |
| `multipart/mixed` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Sequential multipart content |

### Responses

#### Response Object

| Field | Support | Behavior |
|-------|---------|----------|
| `description` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | No longer REQUIRED (was REQUIRED in 3.0/3.1). Absent descriptions handled gracefully |
| `content` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Type changed to `Map[string, Media Type Object \| Reference Object]`. `$ref` values resolved transparently |

### Schema Object

<Note>
No changes from 3.1. See [OAS 3.1 Schema Object](/reference/oas-3-1#schema-object) for full coverage.
</Note>

#### Discriminator Object

| Field | Support | Behavior |
|-------|---------|----------|
| `propertyName` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | 3.2 change: the discriminating property MAY now be optional (was implicitly required in 3.0/3.1). When optional, `defaultMapping` MUST be present |
| `defaultMapping` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Fallback schema when the discriminating property is absent or unmapped |

### Components Object

| Field | Support | Behavior |
|-------|---------|----------|
| `pathItems` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Reusable Path Item Objects via `$ref` from `paths`. Path-level parameters, sibling overrides, and multiple paths referencing the same pathItem all work correctly |
| `mediaTypes` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Reusable Media Type Objects via `$ref` from `content` maps. Referenced objects are resolved and inlined transparently |

### Security Scheme Object

| Field | Support | Behavior |
|-------|---------|----------|
| `deprecated` | <Icon icon="triangle-exclamation" iconType="solid" color="#eab308" size={18} /> | Supported on Operations and Parameters: deprecated items are deprioritized during tool selection. On Schema Object properties: preserved. On Security Schemes: not supported |
| `oauth2MetadataUrl` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | URL to OAuth 2.0 Authorization Server Metadata (RFC 8414) for runtime endpoint discovery |

#### OAuth Flow Object

| Field | Support | Behavior |
|-------|---------|----------|
| `deviceAuthorizationUrl` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | REQUIRED for `deviceAuthorization` flow. Used as the device authorization endpoint |

#### Security Requirement Object

| Feature | Support | Behavior |
|---------|---------|----------|
| URI-based references | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Property names can now be URIs of Security Scheme Objects, enabling cross-document security scheme references. Planned for multi-file specification bundle support |

### Example Object

| Field | Support | Behavior |
|-------|---------|----------|
| `value` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Deprecated in 3.2 for non-JSON serialization targets. Fully functional for all content types |
| `dataValue` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | OAS 3.2 replacement for `value`. Structured example data, valid per the relevant Schema Object. Mutually exclusive with `value` |
| `serializedValue` | <Icon icon="triangle-exclamation" iconType="solid" color="#eab308" size={18} /> | Wire-format string of the example. When `value`/`dataValue` are absent: JSON strings are deserialized and used; non-JSON formats silently skipped |

### Tag Object

| Field | Support | Behavior |
|-------|---------|----------|
| `parent` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Tags with `parent` are resolved to hierarchical slash-separated paths (e.g., tag `refunds` with `parent: payments` becomes `payments/refunds`). Parent chains of any depth with cycle detection. Resolved paths appear in tool comments and metadata.json |
| `kind` | <Icon icon="check" iconType="solid" color="#16a34a" size={20} /> | Classification string (e.g., `nav`, `lifecycle`, `badge`). Rendered alongside tags in tool comments as `tag (kind)` and exposed in metadata.json |
| `summary` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Short summary for display purposes â€” tag names are self-explanatory |

### Encoding Object

| Field | Support | Behavior |
|-------|---------|----------|
| `encoding` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Nested Encoding Objects for recursive encoding in nested multipart content |
| `prefixEncoding` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Positional encoding for multipart (analogous to `prefixItems`) |
| `itemEncoding` | <Icon icon="xmark" iconType="solid" color="#dc2626" size={20} /> | Per-item encoding for multipart arrays (analogous to `items`) |

<Note>
The Encoding Object is not supported. Generated servers send form and multipart bodies using standard encoding â€” per-property overrides (`contentType`, `headers`, `style`) are rarely needed in practice.
</Note>
